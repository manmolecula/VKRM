📁 Структура проекта
В рамках работы проведен сравнительный анализ двух подходов к разработке на примере реального компонента системы (анонимизированный код).

📂 src/SingleComponent.vue — Реализация "ДО" (Vue 2). Монолитный компонент на Options API. Логика, состояние и представление тесно переплетены.
📂 src/MigratedSingleComponent/ — Реализация "ПОСЛЕ" (Vue 3).Модульная архитектура на Composition API.
MigratedSingleComponent.vue — "Тонкий" компонент представления (View).
composables/MigratedSingleComponent.js — Инкапсулированная бизнес-логика (Logic).
📂 tests/ — Сравнительные модульные тесты.

📊 Результаты миграции и метрики
В ходе миграции компонента SingleComponent в архитектуру Vue 3 были получены следующие улучшения:

1. Архитектурные изменения
Характеристика
Vue 2 (Options API)
Vue 3 (Composition API)
Разделение ответственности	Смешение UI и логики в одном файле	Полное разделение (View + Composable)
Переиспользование логики	Сложно (требует Mixins или дублирования)	Легко (импорт функции use...)
Типизация	Сложно (проблемы с типизацией this)	Нативная поддержка TypeScript
Реактивность	Глубокая (Observer) — избыточна для списков	Оптимизированная (shallowRef)

2. Количественные метрики
Сравнение проводилось на анонимизированном коде модуля работы со списками.

Метрика
Vue 2 (Before)
Vue 3 (After)
Динамика
Объем кода компонента (.vue)	~220 строк	~120 строк	Снижение на ~45%
Цикломатическая сложность	Высокая (много ветвлений)	Низкая (разделение функций)	Упрощение поддержки
Объем тестового кода	~30 строк (с учетом стабов)	~12 строк	Сокращение на 60%

3. Качество тестирования (Testing Strategy)
Vue 2 (Integration Testing):
Тестирование требовало создания полноценного экземпляра компонента (mount), мокания глобальных объектов (this.$t, this.$route) и работы с оберткой wrapper.vm.

❌ Медленно: Рендеринг виртуального DOM.
❌ Хрупко: Зависимость от верстки и дочерних компонентов.
Vue 3 (Unit Testing):
Благодаря Composition API мы тестируем чистую JavaScript-функцию MigratedSingleComponent изолированно.

✅ Скорость: Тесты выполняются за миллисекунды.
✅ Изоляция: Не нужно рендерить DOM для проверки бизнес-логики.
✅ Читаемость: Тест выглядит как проверка обычной функции.
🛠 Технологический стек
Framework: Vue 3
Syntax: <script setup>, Composition API
Testing: Vitest, @vue/test-utils
Tooling: Vite, Node.js
